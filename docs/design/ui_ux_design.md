# UI/UX設計書 (ワイヤーフレーム)

## 1. 設計思想

*   `specification.md` の要件に基づき、クリーンで直感的なインターフェースを目指します。
*   Laravel Livewireを全面的に採用するため、ページ遷移を最小限に抑え、インタラクティブな操作性を提供します。
*   レスポンシブデザインを考慮し、デスクトップでもモバイルでも快適に利用できるレイアウトとします。

### 2.2. ダッシュボードページ (`/dashboard`)

```
+--------------------------------------------------+
| (Logo) Dashboard      |   [User Name]▼           |
|                       |   - Settings             |
|                       |   - Language (ja/en)     |
|                       |   - Logout               |
+--------------------------------------------------+
|  +---------------------------+  +----------------+
|  | GitHub Notifications      |  | Add Widget +   |
|  +---------------------------+  +----------------+
|  | - [New] PR #123 needs...  |                   |
|  | - [Read] Issue #456 was.. |  Grid-based layout|
|  | - [New] Mention in...     |  where widgets can|
|  |                           |  be added,        |
|  |       [ Refresh ] [ Settings ]         |  reordered, and   |
|  +---------------------------+  removed.         |
|                                                  |
|  +---------------------------+                   |
|  | Gmail Inbox               |                   |
|  +---------------------------+                   |
|  | - [Unread] John Doe - ... |                   |
|  | - [Unread] Jane Smith -.. |                   |
|  | - [Read] Support - Your.. |                   |
|  |                           |                   |
|  |       [ Refresh ] [ Settings ]         |                   |
|  +---------------------------+                   |
|                                                  |
+--------------------------------------------------+
| Settings | Footer Links                          |
+--------------------------------------------------+
```
*   **インタラクション:**
    *   `[Add Widget +]` ボタンをクリックすると、利用可能なウィジェット一覧がモーダルで表示されます。
    *   ウィジェットはドラッグ＆ドロップで並べ替え可能です。
    *   各ウィジェットの `[Refresh]` ボタンは、Livewireのアクション (`wire:click="refresh"`) に対応し、そのウィジェットのみを非同期で更新します。
    *   `[Settings]` ボタンをクリックすると、そのウィジェットの設定モーダルが開きます。

#### 2.2.1. ウィジェット設定モーダル

```
+--------------------------------------------------+
|  GitHub Notifications Settings                   |
+--------------------------------------------------+
|  +-----------+ +-----------+                     |
|  | General   | | Sharing   |                     |
|  +-----------+ +-----------+                     |
|                                                  |
|  +---------------------------------------------+ |
|  |  Sharing Settings                           | |
|  +---------------------------------------------+ |
|  |                                             | |
|  |  (o) Private (only me)                      | |
|  |  ( ) Share with specific users              | |
|  |  ( ) Share with a group                     | |
|  |  ( ) Share with specific servers            | |
|  |                                             | |
|  |  Group: [ Group A ▼]                        | |
|  |          - Group B                         | |
|  |                                             | |
|  |  +----------------------------------------+ | |
|  |  | Enter usernames (comma separated)      | | |
|  |  +----------------------------------------+ | |
|  |                                             | |
|  |  Shareable Link: [ https://.../s/token ] [Copy] |
|  |                                             | |
|  +---------------------------------------------+ |
|                                                  |
|  +-----------------+  +-----------------+        |
|  |      Save       |  |     Cancel      |        |
|  +-----------------+  +-----------------+        |
+--------------------------------------------------+
```
*   **インタラクション:**
    *   `Sharing` タブで公開範囲を選択します。
    *   「Share with specific users」を選択すると、ユーザー名を入力するテキストボックスが表示されます。
    *   「Share with specific servers」を選択すると、サーバー識別子を入力するテキストボックスが表示されます。
    *   設定を保存すると、`sharing_settings` と `sharing_rules` テーブルが更新されます。
    *   `Shareable Link` は、外部からデータにアクセスするためのURLです。

### 2.3. 設定ページ (`/settings`)

タブ形式で各種設定項目を分類します。

#### 2.3.1. 連携サービス管理タブ (`/settings/accounts`)

```
+--------------------------------------------------+
| (Logo) Dashboard      |   [User Name]▼           |
|                       |   - Settings             |
|                       |   - Language (ja/en)     |
|                       |   - Logout               |
+--------------------------------------------------+
|  Settings                                        |
|  +-----------+ +----------+ +----------+ +--------------------+ +----------+ +----------+
|  | Profile   | | Groups   | | Accounts | | Server Connections | | Language | | Appearance|
|  +-----------+ +----------+ +----------+ +--------------------+ +----------+ +----------+
|                                                  |
|  +---------------------------------------------+ |
|  |  Connected Accounts                         | |
|  +---------------------------------------------+ |
|  |                                             | |
|  |  GitHub        [ Status: Connected ] [Revoke]| |
|  |  ------------------------------------------- | |
|  |  Google        [ Status: Connected ] [Revoke]| |
|  |  ------------------------------------------- | |
|  |  Twitter (X)   [ Status: Not Connected ] [Connect] |
|  |                                             | |
|  +---------------------------------------------+ |
|                                                  |
+--------------------------------------------------+
```
*   **インタラクション:**
    *   `[Connect]` ボタンをクリックすると、各サービスのOAuth認証ページにリダイレクトします。
    *   `[Revoke]` ボタンをクリックすると、確認モーダルを表示した後、Livewireのアクションで連携を解除し、関連する `connected_accounts` レコードを削除します。

## 3. スタイリング

*   CSSフレームワークとして **Tailwind CSS** の利用を想定します。
*   `specification.md` にある通り、Node.jsのサーバー環境は必須としないため、開発環境でTailwind CSSをビルドし、生成されたCSSファイルを本番環境にデプロイする運用とします。これにより、共有サーバーへの設置が容易になります。
